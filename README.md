# Nexenova Studios Website

The marketing website for Nexenova Studios — a mobile game development studio based in India.

Built with Vite + React 18 + Tailwind 4. Includes a public marketing site (hero, about, services, portfolio, team, contact, legal pages) and an admin dashboard at `/admin` for authorized personnel to manage content via Supabase.

## Stack

- **Framework**: Vite 6 + React 18 + TypeScript
- **Styling**: Tailwind 4, Radix UI primitives, MUI, framer-motion
- **Backend**: Supabase (Postgres + Edge Functions in `src/app/supabase/functions/server/`)
- **Deploy**: Cloudflare Pages (see `wrangler.toml`)

## Local development

Requires Node 18+ and npm.

```bash
npm install
npm run dev
```

The dev server runs on http://localhost:5173/.

> **Note on imports**: this codebase was originally generated by Figma Make and uses non-standard `pkg@version` style imports. A Vite resolver plugin in `vite.config.ts` (`figmaVersionedImportResolver`) maps them to the real packages at import time — do not remove it.

## Build for production

```bash
npm run build
```

Static output is written to `dist/`. Cloudflare Pages picks this up via `wrangler.toml`.

## Admin dashboard

`/admin` (or `#admin`) shows the login page. Authorized personnel can manage:

- Games portfolio (CRUD with image uploads)
- Site content: hero stats, team members, company info
- Users and audit log

Until you configure your own Supabase project (see below), admin login will not persist changes — the UI falls back to bundled defaults in `src/app/components/admin/DataManager.ts`.

### Wiring your own Supabase project

1. Create a Supabase project at https://supabase.com.
2. Replace `projectId` and `publicAnonKey` in `src/app/utils/supabase/info.tsx` with your project's values.
3. Deploy the Edge Function in `src/app/supabase/functions/server/index.tsx` to your project.
4. Set the following secrets in Supabase: `JWT_SECRET`, `ENCRYPTION_KEY`, `SUPABASE_SERVICE_ROLE_KEY`, `SUPABASE_URL`.
5. The function will create a default admin on first run — credentials shown on the login page (change immediately).

## Deployment

Cloudflare Pages is set up via `wrangler.toml`:

- **Build command**: `npm run build`
- **Output directory**: `dist`

Connect this repo in the Cloudflare Pages dashboard, or deploy manually with `npx wrangler pages deploy dist`.

SPA fallback is handled by `public/_redirects` (`/* /index.html 200`).
